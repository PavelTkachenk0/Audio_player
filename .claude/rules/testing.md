# Тесты

xUnit + EF Core InMemory. Проект `Audio_player.Tests`. Запуск: `dotnet test`.

- Предпочитать `[Theory]` + `[InlineData]` вместо множества `[Fact]` (одна логика — разные данные).
- **DB-тесты разнесены на 3 слоя** (в подпапке по фиче):
  - `{X}Tests` — тонкий, только Arrange-Act-Assert, данные через `[MemberData]`.
  - `{X}TestCases` — `IEnumerable<object[]>` наборы входов.
  - `{X}TestData` — билдеры InMemory-БД и сущностей (переиспользуемо).
- Простые юнит-тесты (хешер паролей, токены) — `[InlineData]` рядом с методом, без разноса (разнос был бы церемонией).
- Тесты пишутся ДО рефактора как страховка от регрессий (фиксируют текущее поведение).
- Граничные кейсы приветствуются (пустые наборы, разные пользователи).
